1057数零壹

一天没写博客了，今天来更新一下。主要原因是1051、1053、1056简单一些，1054太狗了一些，1050肝了两个多小时没肝出来，所以还是来写一下1057这道题吧。

本题较为正常。

题目

给定一串长度不超过 105 的字符串，本题要求你将其中所有英文字母的序号（字母 a-z 对应序号 1-26，不分大小写）相加，得到整数 N，然后再分析一下 N 的二进制表示中有多少 0、多少 1。例如给定字符串 PAT (Basic)，其字母序号之和为：16+1+20+2+1+19+9+3=71，而 71 的二进制是 1000111，即有 3 个 0、4 个 1。

输入格式

输入在一行中给出长度不超过 105、以回车结束的字符串。

输出格式

在一行中先后输出 0 的个数和 1 的个数，其间以空格分隔。注意：若字符串中不存在字母，则视为 N 不存在，也就没有 0 和 1。

本题思路

先创建一个数组，用来表示字母序号，这里用动态数组循环26次插入从1~26的数。其次，将字符串输入进去，这里要注意，它要求输入字符串是以回车结束，即字符串之间可能有空格，这里我们可以用getline(cin,s);。但是我傻傻的先建了一个string类型的动态数组，并且把其中插入了string元素，再将它们连接起来，这也导致了我初始代码跑起来永远是第四个测试点无法通过(代码超时)。

然后我们对string进行操作，对每一个元素进行判断大小写，并且把序号加到sum里面即可。

最后将sum用短除法化为二进制，如果除出来为1，cnt1就自加，除出来为0，cnt0就自加。
